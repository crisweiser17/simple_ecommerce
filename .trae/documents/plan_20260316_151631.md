This significantly expands the scope. We are moving from a static catalog to a **Dynamic Web Application** with a database and authentication.

### **The New Architecture**

#### **1. Database (SQLite)**
To keep it "simple, agile, and efficient" without complex server configuration, we will use **SQLite**. It is a serverless database stored as a single file (`database.sqlite`), perfect for this project.
*   **`products` Table:** ID, Name, SKU, Price, Image, Description, Category, PDF URL.
*   **`orders` Table:** ID, Customer Info (JSON), Items (JSON), Date.
*   **`users` Table:** Email, Login Token, Token Expiry.

#### **2. Authentication (Magic Link/Token)**
*   **Login Flow:** User enters Email -> System generates a **6-digit code** -> System sends code (simulated locally) -> User enters code -> Logged in.
*   **Admin Access:** We will designate a specific email (e.g., `admin@r2.com`) as the administrator.

#### **3. Admin Panel (`/admin`)**
A simple, password-protected dashboard for you to manage the business.
*   **Products Manager:** Add, Edit, and Delete products (upload images/PDFs).
*   **Order Viewer:** View all saved orders with customer details (Name, WhatsApp, Address).

#### **4. Customer Checkout Flow**
1.  **Cart:** Review items.
2.  **Identity:** Enter Email (Login/Register via Token).
3.  **Details:** Form for Name, WhatsApp, Address.
4.  **Finish:** Save Order to Database -> **Generate PDF Quote** -> Display Success.
