Vou realizar as seguintes alterações para corrigir o erro do PDF e implementar a funcionalidade de envio para WhatsApp:

### 1. Correção do Erro Dompdf
- **Ação**: Modificar `src/generate_pdf.php` para remover a definição explícita da fonte 'Helvetica', permitindo que o Dompdf use sua fonte padrão (serif) ou 'DejaVu Sans'. Isso resolve o erro de "Undefined array key" relacionado às métricas de fonte corrompidas ou ausentes.

### 2. Configuração do WhatsApp no Admin
- **Banco de Dados**: Criar uma tabela `settings` para armazenar configurações globais, como o número do WhatsApp da loja.
- **Backend**: Adicionar funções `getSetting` e `updateSetting` em `src/functions.php`.
- **Painel Admin**: Adicionar uma nova aba "Configurações" em `templates/admin/dashboard.php` com um campo para editar o número do WhatsApp que receberá os pedidos.
- **Rota**: Criar a rota `/admin/save-settings` em `index.php` para salvar essa configuração.

### 3. Novo Fluxo de Checkout com Botão WhatsApp
- **Alteração no Checkout**: Modificar a rota `/checkout` em `index.php` para, ao invés de baixar o PDF imediatamente e encerrar, redirecionar o usuário para uma nova página de sucesso (`/order-success?id=...`).
- **Página de Sucesso (`templates/order-success.php`)**: Criar esta nova página que exibirá:
    - Resumo do pedido.
    - Botão **"Baixar PDF do Pedido"** (aponta para uma nova rota `/download-pdf`).
    - Botão **"Enviar Pedido via WhatsApp"** (gera um link `wa.me` com a mensagem formatada contendo os itens e dados do cliente para o número configurado no admin).
- **Rota de Download**: Criar a rota `/download-pdf` em `index.php` que chama a função `generateOrderPDF` existente.

Este plano separa a geração do PDF da finalização do pedido, permitindo a interação com o WhatsApp e resolvendo o erro de renderização.
