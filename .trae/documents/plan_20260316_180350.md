I will implement a lightweight internationalization (i18n) system to support both English and Portuguese.

### 1. Create Language Infrastructure
- **Create `src/i18n.php`**: This file will handle:
    - Language selection logic (checking URL `?lang=pt` or Session).
    - Loading the appropriate translation file.
    - Defining the global helper function `__($key)` to retrieve translations.
- **Create Translation Files**:
    - `lang/en.php`: PHP array returning English strings.
    - `lang/pt.php`: PHP array returning Portuguese strings.

### 2. Integrate into Application
- **Update `index.php`**: Include `src/i18n.php` at the very beginning so translations are available globally.
- **Update `src/functions.php`**: Ensure no conflicts, though the helper will likely live in `i18n.php`.

### 3. Add Language Switcher to UI
- **Update `templates/layout.php`**: Add a language selector (e.g., "🇺🇸 EN | 🇧🇷 PT") in the top navigation bar.
- This selector will link to `?lang=en` and `?lang=pt`.

### 4. Apply Translations
- **Refactor Templates**: I will go through the main templates (`layout.php`, `login.php`, `cart.php`, `archive.php`) and replace hardcoded text with the `__('Key')` function.
    - *Example*: Change `Sign In` to `__('Sign In')`.

### 5. Verification
- I will verify by clicking the language toggle and ensuring the interface text switches between English and Portuguese.

**Note**: This plan covers static interface text (menus, buttons, labels). Dynamic database content (like product names) will remain in their original language unless we decide to restructure the database later.
